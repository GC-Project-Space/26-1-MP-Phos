# Product Docs Index: Phos

이 문서는 `docs/product` 아래의 제품 문서를 어떤 순서로 읽으면 좋은지 안내하는 인덱스입니다.

## 추천 읽기 순서

1. [`PRD-Phos.md`](./PRD-Phos.md)
   - 제품 목표, MVP 범위, 핵심 사용자 흐름과 기능을 정의합니다.
2. [`UI-UX-REQUIREMENTS-Phos.md`](./UI-UX-REQUIREMENTS-Phos.md)
   - PRD와 backlog를 실제 모바일 화면 흐름과 UX 요구사항으로 재정리합니다.
3. [`DESIGN-SYSTEM-Phos.md`](./DESIGN-SYSTEM-Phos.md)
   - MVP 모바일 UI의 시각 원칙, 토큰, 상호작용 규칙을 정의합니다.
4. [`COMPONENT-DESCRIPTIONS-Phos.md`](./COMPONENT-DESCRIPTIONS-Phos.md)
   - 현재/예정 컴포넌트의 역할과 상태, 데이터 의존성을 정리합니다.
5. [`USER-STORIES-Phos.md`](./USER-STORIES-Phos.md)
   - PRD를 실제 구현 가능한 사용자 스토리 단위로 풀어냅니다.
6. [`WWA-Backlog-Phos.md`](./WWA-Backlog-Phos.md)
   - Why-What-Acceptance 형식으로 backlog를 정리합니다.
7. [`API-SPEC-Phos.md`](./API-SPEC-Phos.md)
   - 세션 중심 API 계약과 리소스 구조를 정의합니다.
8. [`EVENT-SCHEMA-Phos.md`](./EVENT-SCHEMA-Phos.md)
   - 제품/운영/프라이버시 이벤트 정의를 정리합니다.
9. [`TEST-SCENARIOS-Phos.md`](./TEST-SCENARIOS-Phos.md)
   - MVP QA 시나리오와 검증 범위를 정리합니다.
