## 1) 요약 (1단계)

이 문서는 `Phos`의 MVP v1을 만들기 위한 제품 요구사항 문서입니다.  
Phos는 짧은 시간 안에 촬영, 간단 편집, 저장까지 끝내는 재미 중심 모바일 인생네컷 앱입니다.  
첫 버전은 `Live Booth`를 중심으로 만들고, 사진과 메이킹 영상을 모두 결과물로 제공하는 데 집중합니다.

## 3) 배경 (3단계)

### 맥락

기존 인생네컷 경험은 오프라인 포토부스에 묶여 있거나, 모바일에서는 사진 편집만 강한 경우가 많습니다.  
Phos는 모바일에서 바로 인생네컷 촬영을 하고, 촬영 과정의 메이킹 영상까지 함께 남기게 하려는 제품입니다.  
핵심은 `실용성`보다 `재미`입니다.

### 지금 이 시점인 이유

- 모바일 카메라와 영상 처리 경험이 충분히 익숙해졌습니다.
- 앱 내에서 결과물을 즉시 저장/확보하려는 사용 패턴이 일반화되었습니다.
- 프라이버시 민감도가 높아져, `계정 없이 쓰고 빠르게 지워지는` 경험이 차별점이 될 수 있습니다.

### 달라진 점

- 사진만 잘 나오는 앱보다, `찍는 순간의 에너지`까지 남기는 영상형 결과물이 더 매력적인 시대가 되었습니다.
- 사용자는 설치와 가입보다 `바로 찍고 바로 받는 흐름`을 더 선호합니다.

## 4) 목표 (4단계)

### 목표

- **PRD-01**: 사용자가 2분 안에 촬영을 끝내고, 최종 포토 스트립 또는 메이킹 영상을 자기 기기에 확보하게 만드는 것입니다.
- **PRD-02**: 이 경험이 빠르고 재미있어야 다시 쓰고 다른 사람에게 보여줄 이유가 생깁니다.

### 비목표 (Non-Goals / Out of Scope)

- **PRD-NG-01**: 고급 사진 편집 도구
- **PRD-NG-02**: 복잡한 템플릿 커스터마이징
- **PRD-NG-03**: 계정 기반 개인화 흐름
- **PRD-NG-04**: `Album Edit` 본 기능의 MVP 포함
- **PRD-NG-05**: 출시 게이트로서의 고도화된 세션 복구

### 왜 중요한가

- 사용자 입장: 복잡한 편집 없이도 특별한 순간을 바로 결과물로 남길 수 있습니다.
- 제품 입장: `사진 + 메이킹 영상` 조합은 차별화된 핵심 경험입니다.
- 사업 입장: 결과물 확보 경험이 안정적일수록 재방문과 추천 가능성이 커집니다.

### 전략 정렬

- 제품 전략: 재미 중심, 빠른 완주, 결과물 확보, 프라이버시 신뢰
- 기술 전략: 세션 중심 구조, 표준 API 우선, 낮은 기기 성능까지 고려한 안정성

### 핵심 결과 (SMART)

| ID            | KR  | 정의                           | 목표                                  |
| ------------- | --- | ------------------------------ | ------------------------------------- |
| **PRD-KR-01** | KR1 | 첫 세션 완주율                 | >= 70%                                |
| **PRD-KR-02** | KR2 | 결과물 확보 전환율             | >= 80%                                |
| **PRD-KR-03** | KR3 | 다운로드 성공률                | >= 97%                                |
| **PRD-KR-04** | KR4 | 세션 실패율                    | < 3%                                  |
| **PRD-KR-05** | KR5 | 2-Minute Keepsake Secured Rate | 소프트 런치 2주차 >= 30%, D+30 >= 45% |

## 5) 시장 세그먼트 (5단계)

### 주요 세그먼트

- **PRD-MS-01**: `짧은 시간 안에 분위기 있는 포토 스트립을 만들고 싶은 사용자`
  - 친구와 노는 중 빠르게 사진을 남기고 싶을 때
  - 데이트, 파티, 여행, 행사에서 추억을 바로 만들고 싶을 때
  - 따로 편집 앱을 열지 않고 한 번에 끝내고 싶을 때

### 보조 세그먼트

- **PRD-MS-02**: `행사나 모임에서 여러 사람이 쉽게 결과물을 받고 싶은 사용자`
  - 생일, 웨딩, 소규모 이벤트에서 사진/영상을 빠르게 확보하고 싶을 때
  - 참가자가 계정 없이 결과물을 받고 싶을 때

### 제약 조건

- **PRD-CON-01**: 저사양 기기에서도 동작해야 함
- **PRD-CON-02**: 촬영 흐름은 2분 안에 끝나야 함
- **PRD-CON-03**: 계정 없이 세션 사용 가능해야 함
- **PRD-CON-04**: 48시간 보관 후 자동 삭제되어야 함
- **PRD-CON-05**: 사용자는 언제든 삭제/내보내기를 요청할 수 있어야 함

## 6) 가치 제안 (6단계)

### 사용자가 하려는 일

- 재미있는 방식으로 사진을 찍고 싶다
- 결과물을 빠르게 저장하고 확보하고 싶다
- 촬영 순간의 분위기까지 남기고 싶다
- 개인 사진이 오래 남거나 다른 용도로 쓰이지 않기를 바란다

### 기대 이득

- **PRD-VP-01**: 사진과 영상 두 결과물을 한 번에 얻는다
- **PRD-VP-02**: 복잡한 편집 없이도 인생네컷 스타일 결과를 만든다
- **PRD-VP-03**: 앱 내 결과 화면에서 빠르게 저장한다
- **PRD-VP-04**: 가입 부담 없이 사용한다

### 제거하려는 불편

- 촬영 앱과 편집 앱을 오가는 번거로움
- 저장 경로가 느리거나 복잡해서 흐름이 끊기는 문제
- 프라이버시 정책이 불명확한 불안감
- 저사양 기기에서 생기는 버벅임과 크래시

### 차별화 포인트

- `최종 포토 스트립 + 메이킹 영상`을 기본 경험으로 묶는다
- `빠른 완주`를 핵심 가치로 둔다
- `계정 없이 사용 + 짧은 보관 + 즉시 삭제/내보내기`를 제품 가치로 드러낸다

## 7) 해결안 (7단계)

### 7.1 UX / 프로토타입

핵심 사용자 흐름은 [UI-UX 요구사항](./UI-UX-REQUIREMENTS-Phos.md)에서 상세히 정의합니다. PRD 수준의 주요 단계는 다음과 같습니다.

1. 앱 진입 -> 프레임 선택
2. Live Booth 촬영 (카운트다운 기반 4/6컷)
3. 촬영 중 메이킹 영상 자동 기록
4. 컷 선택 및 간단 편집
5. 최종 포토 스트립 렌더링 및 결과 확인
6. 로컬 저장 및 세션 종료 (또는 삭제/내보내기 요청)

UX 원칙:

- 메인 흐름은 한 손으로도 이해 가능해야 함
- 주요 액션은 `촬영`, `다음`, `저장` 수준으로 단순해야 함
- 프라이버시 정보는 짧고 분명하게 노출함

### 7.2 핵심 기능

상세 구현 요건은 [사용자 스토리](./USER-STORIES-Phos.md) 및 [WWA 백로그](./WWA-Backlog-Phos.md)를 참조하십시오.

#### **PRD-F-01**: 프레임 선택

- 사용자는 촬영 전 4컷, 6컷 중심의 기본 프레임을 선택할 수 있다.

#### **PRD-F-02**: Live Booth 촬영

- 사용자는 카운트다운 기반으로 연속 촬영을 진행하며, 앱은 세션 단위로 상태를 유지한다.

#### **PRD-F-03**: 메이킹 영상 기록

- 촬영 시작과 동시에 메이킹 영상을 기록하며, 이는 최종 포토 스트립과 별도의 결과물로 제공된다.

#### **PRD-F-04**: 간단 편집

- 촬영 후 컷 선택, 순서 변경, 기본 필터/텍스트/크롭 기능을 제공한다.

#### **PRD-F-05**: 최종 렌더링

- 선택된 컷과 프레임으로 최종 포토 스트립을 생성하며, 결과는 세션 단위로 관리된다.

#### **PRD-F-06**: 로컬 저장

- 사용자는 최종 결과물(사진/영상)을 자기 기기에 저장할 수 있다.

#### **PRD-F-07**: 프라이버시 제어

- `trainingOptIn=false` 기본값, 삭제/내보내기 요청 기능을 제공한다.

#### **PRD-F-08**: 세션 복구 (안정화 항목)

- 비정상 종료 후 재진입 시 미완료 세션 복구를 시도한다. (V1 이후 품질 강화 항목)

#### **PRD-F-09**: Album Edit (후속 우선순위)

- 기존 사진을 프레임에 넣어 편집하는 기능 (V1.5 예정)

### 7.3 기술 방향

상세 명세는 [API 명세](./API-SPEC-Phos.md) 및 [이벤트 스키마](./EVENT-SCHEMA-Phos.md)를 참조하십시오.

- **세션 중심 구조**: 모든 활동은 익명 `sessionId` 기반의 세션으로 관리됨
- **리소스 모델**: `Session`, `Asset`, `Frame`, `Consent`, `ExportRequest`, `DeletionRequest`
- **데이터 보관**: `retentionExpiresAt` 기준 48시간 자동 삭제 불변조건 준수
- **성능 최적화**: 저사양 기기 대응을 위한 해상도/처리량 자동 조절

### 7.4 가정

- 사용자는 사진만이 아니라 메이킹 영상도 가치 있게 느낀다.
- 2초 내 카메라 진입이 완주율을 높인다.
- 48시간 보관과 분리 동의는 신뢰를 높이고 큰 이탈을 만들지 않는다.

## 8) 출시 계획 (8단계)

### 버전 계획

#### V1 (MVP)

- Live Booth 촬영, 메이킹 영상 기록, 간단 편집, 최종 렌더링, 로컬 저장, 48시간 자동 삭제, 프라이버시 제어

#### V1.5+

- Album Edit, 프레임 카탈로그 확장, 세션 복구 안정성 개선

### 출시 게이트

검증 시나리오는 [테스트 시나리오](./TEST-SCENARIOS-Phos.md)를 참조하십시오.

- 핵심 결과(KR) 지표 충족 (KR1~KR5)
- 세션 크래시 비율 < 2.0%, 영상 기록 실패 비율 < 3.0%
- 만료 자산 접근 비율 = 0%, 삭제/내보내기 SLA 충족
- 프라이버시/다운로드 신뢰성 실패 시 출시 보류
