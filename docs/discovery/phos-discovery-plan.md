# 디스커버리 계획: Phos (모바일 인생네컷 포토부스)

**날짜**: 2026-03-20  
**제품 단계**: 신규 제품 (초기 검증 단계)  
**디스커버리 질문**: `재미 중심` 인생네컷 경험에서, 사용자가 2분 내 촬영-편집-저장을 끝내고 사진/메이킹 영상을 모두 확보하도록 만드는 가장 강한 MVP는 무엇인가?

---

## 1) 디스커버리 맥락 (1단계)

### 우리가 이미 아는 것

- 목표: 실용성보다 `재미`와 `짧은 완주 경험`이 핵심.
- 핵심 모드: `Live Booth` (연속 촬영 + 메이킹 영상 기록) / `Album Edit` (기존 사진으로 프레임 편집).
- MVP v1 필수: 프레임 선택, Live Booth 촬영, 메이킹 영상 기록, 컷 선택/간단 편집, 최종 렌더링, 앱 내 결과 화면 확인, 로컬 저장, 48시간 자동 삭제, 서비스 이용 동의와 선택적 데이터 활용 동의 분리, 즉시 삭제/내보내기.
- 프라이버시/안전 방향: trainingOptIn 기본 OFF, 최소 수집, 짧은 보관(48시간), 서비스 이용 동의와 선택적 데이터 활용 동의 분리, 즉시 삭제/내보내기.
- API 방향: Google AIP-121 기반 리소스 지향 + 표준 메서드 우선, 필요한 경우에만 커스텀 메서드.

### 아직 모르는 것 (이번 디스커버리에서 검증)

- 사용자가 가장 강하게 반응하는 `재미 트리거`는 무엇인가? (프레임/타이머/사운드/메이킹 영상)
- 메이킹 영상 기록이 실제 공유율을 얼마나 올리는가?
- 앨범 내 기존 프레임 사진 자동 수집 기능의 정확도/체감 가치가 충분한가?
- 2초 내 촬영 진입과 저사양 대응이 사용자 만족 임계치를 넘는가?

### 이번 디스커버리가 알려줄 의사결정

- 채택/중단: 메이킹 영상 다운로드를 MVP 핵심으로 고정할지
- 우선순위 결정: 자동 수집 기능 vs 편집 고도화 vs 결과물 확보 기능 우선순위
- 방향 전환: 신규 촬영 중심 앱 vs 앨범 리믹스 중심 앱

### 실험 상세

#### E1. 결과물 가치 검증

- **가설**: 메이킹 영상을 함께 제공하면 실제 저장 행동이 증가한다.
- **구성**: 사용자 본인 폰에서 결과 화면 진입 후 사진/영상 저장까지 수행.
- **측정**: 2분 내 저장 완료율, 저장 실패 지점.
- **의사결정**: 기준 미달 시 영상 기본 ON 대신 토글형으로 전환.

#### E2. 2초 진입 퍼널 검증

- **가설**: 촬영 진입시간 단축이 완주율을 올린다.
- **구성**: 프로토타입 기반 태스크 테스트(Quick Start vs 일반 진입), 트래픽 확보 후 A/B.
- **측정**: time-to-camera, 진입-촬영완료-저장완료 전환률.
- **의사결정**: 2초 달성률/완주율 기준 미달 시 진입 UI 재설계 우선.

#### E3. 저사양 안정성 스파이크

- **가설**: 해상도 자동 조절로 저사양에서도 메이킹 영상 기록 가능.
- **구성**: 해상도 프리셋(높음/중간/낮음) 자동 전환 로직 시험.
- **측정**: 세션 실패율, 메모리 피크, 체감 지연.
- **의사결정**: 실패율 3% 초과 시 메이킹 영상 기록 품질 단계별 제한 도입.

#### E4. 자동 수집 정확도 검증

- **가설**: 규칙 기반 분류로 프레임 사진 자동 식별 가능.
- **구성**: 앨범 샘플 라벨셋 구축 후 룰 기반 분류기 측정.
- **측정**: precision/recall, 사용자 체감 유용성.
- **의사결정**: 정확도 미달 시 즉시 사용자 확인형 UX로 축소하고 후속 우선순위로 이관.

#### E5. 프라이버시 신뢰 UX 검증

- **가설**: 분리 동의 + 48시간 자동 삭제 명시가 신뢰를 높인다.
- **구성**: 동의 UI 2종 비교(기본형 vs 투명성 강화형).
- **측정**: 정책 이해도, 삭제/내보내기 태스크 완료율, 이탈률.
- **의사결정**: 이해도/완료율 기준 미달 시 카피 재작성 + 정보 구조 단순화.

#### E6. 결과물 저장 안정성 검증

- **가설**: 만료 안내/재시도 UX가 저장 성공률을 유지한다.
- **구성**: 3G/불안정 Wi-Fi 포함 네트워크 시나리오 테스트.
- **측정**: 재시도 성공률, 총 완료시간, 에러 회복률.
- **의사결정**: 성공률 미달 시 링크 만료 정책/재시도 플로우/파일 분할 전략을 우선 개선.

## 7) API/프라이버시 구체화 (검색 결과 반영)

### A) 리소스 지향 API (AIP-121 계열 반영)

- 표준 메서드 우선: `Get/List/Create/Update/Delete`
- 커스텀 메서드는 꼭 필요한 동작에만 사용
  - `POST /v1/sessions/{sessionId}:render`
  - `POST /v1/sessions/{sessionId}:finalize`

권장 리소스 트리 (MVP):

- `Session`: `/v1/sessions`, `/v1/sessions/{sessionId}`
- `Asset`: `/v1/sessions/{sessionId}/assets`, `/v1/sessions/{sessionId}/assets/{assetId}`
- `Frame`: `/v1/frames`, `/v1/frames/{frameId}`
- `Consent`: `/v1/sessions/{sessionId}/consents`, `/v1/sessions/{sessionId}/consents/{consentId}`
- `ExportRequest`: `/v1/sessions/{sessionId}/exportRequests`, `/v1/sessions/{sessionId}/exportRequests/{exportRequestId}`
- `DeletionRequest`: `/v1/sessions/{sessionId}/deletionRequests`, `/v1/sessions/{sessionId}/deletionRequests/{requestId}`

응답 메타(요구사항 고정):

- `retentionExpiresAt`
- `trainingUsed`
- `consentVersion`
- `deletionStatus`

### B) 프라이버시/안전 운영 가드레일

- 기본값: `trainingOptIn=false`
- 최소 수집: 계정 없이 세션 생성 가능(익명 sessionId)
- 보관: 기본 48시간 자동 삭제, 목적/만료시간 명시
- 서비스 이용 동의와 선택적 데이터 활용 동의 분리
- 사용자 권리: 즉시 삭제 요청, 데이터 내보내기 요청
- 삭제/보관 불변조건:
  - `retentionExpiresAt` 이후 결과물 접근 차단
  - 세션 삭제 즉시 해당 세션 자산 접근 차단
  - 삭제 확정 전 내보내기 요청 가능, 내보내기 완료 후에도 보관 규칙 동일 적용
- 로그 안전성: 해시/내부 ID, 시각, 액션 타입, consentVersion만 보관(원본 URL/PII 미보관)
- 학습 활용 보장: `trainingUsed=true`는 해당 세션의 동의 스냅샷(`consentVersion`)이 있을 때만 허용

프라이버시 상태 전이:

- `active -> export_requested -> delete_requested -> deleted`

## 8) 디스커버리 일정 (7단계)

- **Week 1**: E1, E3 실행 (핵심 가치/기술 가능성 동시 검증)
- **Week 2**: E2, E4, E5 실행 (퍼널/자동수집/프라이버시 UX)
- **Week 3**: E6 + 통합 분석 + 채택/중단/방향 전환 의사결정

## 9) 의사결정 프레임워크 (7단계)

- If `E1+E2` 성공 -> Live Booth 중심 MVP 확정, 영상 기본 제공 유지
- If `E3` 실패 -> 저사양 모드 기본값 강화(해상도/프레임레이트 제한)
- If `E4` 실패 -> 자동 수집 기능은 후속 우선순위로 이관, 반자동 추천 UX만 유지
- If `E5` 성공 -> 분리 동의/48시간 자동 삭제 정책을 온보딩 핵심 메시지로 전면 배치
- If `E6` 실패 -> 만료 정책/저장 재시도 플로우 우선 개선
- If `E5` 또는 `E6` 실패 -> 출시 게이트 보류(프라이버시/저장 신뢰성 미충족)

최종 게이트(Go 조건):

- 완주율 >= 70%
- 세션 실패율 < 3%
- 다운로드 성공률 >= 97%
